# Legacy RPG — Design Spec

**Date:** 2026-03-27

## Vision

A text-based multiplayer RPG set in a great, magical wood. Players wander through a living forest, discovering locations, meeting NPCs, and finding magical items. Every action by any player has lasting, permanent consequences on the shared world. The world runs indefinitely, shaped by its history.

---

## Architecture

```
┌─────────────────────────────────────────────┐
│  CLI client (Node.js)                        │
│  - Reads typed input                         │
│  - Streams responses to terminal             │
│  - Connects to API server via HTTP           │
└──────────────────┬──────────────────────────┘
                   │ HTTP (localhost)
┌──────────────────▼──────────────────────────┐
│  Game API server (Node.js + Express)         │
│  - Intent parser  → Cloudflare AI            │
│  - NPC dialogue   → Venice (GLM-5)           │
│  - World changes  → Claude                   │
│  - State reads/writes → SQLite               │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│  SQLite database (local)                     │
│  - world state, locations, NPCs, items       │
│  - event log (every consequence recorded)    │
│  - season/time tracking                      │
└─────────────────────────────────────────────┘
```

- **CLI** is a thin client — it sends raw input to the API and streams responses to the terminal.
- **API server** handles all game logic, AI routing, and database writes.
- **SQLite** stores all world state locally on the server.
- The CLI/API separation means a browser frontend (web terminal) can be added later with zero backend changes.

## World Model

The world is a graph of **locations** connected by paths. Locations are generated by AI on first discovery and persist forever, mutated by events.

### Entities

| Entity | Key fields |
|---|---|
| **Location** | name, description, atmosphere, connections, seasonal state |
| **NPC** | name, personality, current location, interaction memory, per-player relationship |
| **Item** | name, description, magical properties, current location, ownership history |
| **Player** | username, current location, inventory, action history |
| **Event log** | who, what, where, when (game-time), what changed — immutable, append-only |
| **Season** | current season (Spring/Summer/Autumn/Winter), advances on real-time clock |

### Key design decision — the event log

Every world-altering action is appended to an immutable event log. This is the world's memory. AI prompts include relevant recent history so NPCs and consequence reasoning are always grounded in real past events, not just current state.

## AI Routing

Three models, each doing what it's best at:

### Cloudflare AI — Intent parsing
Every player input is classified first: movement, dialogue, item interaction, world action, or ambiguous. Fast and cheap — runs on every input before any other processing.

### Venice / GLM-5 — NPC dialogue and atmosphere
- NPC responses in character, with personality, memories, and current mood as context
- Atmospheric location descriptions on entry
- Creative, uncensored, low latency

### Claude — Consequence reasoning
- Only invoked when an action may change the world
- Receives full local event history as context
- Returns structured JSON describing what changes (location mutations, item moves, NPC state updates)
- Server applies the changes to SQLite

**Cost principle:** Most interactions (movement, dialogue) never touch Claude. Only meaningful world-altering actions do.

## CLI Experience

### Session flow
1. Launch CLI → atmospheric intro + current location description (streamed)
2. Prompt: `> `
3. Player types natural language freely
4. Response streams back — narration, NPC dialogue, world reactions

### UX principles
- **Streaming** — text appears word by word, typewriter-style
- **Location header** — always visible: current location, season, time of day
- **No commands to memorize** — pure natural language throughout
- **Multiplayer bleed** — world occasionally surfaces nearby events caused by other players ("you notice fresh tracks", "smoke rises to the north")
- **Graceful ambiguity** — unclear intent triggers a clarifying question, never a silent failure or wrong action

### Browser extension path
Later: a web terminal (xterm.js or similar) connects to the same API. Visually and functionally identical to the CLI experience. No backend changes required.

## Seasons

- Four seasons: Spring, Summer, Autumn, Winter
- Advance on a real-time clock (cadence TBD during implementation)
- Affect: location descriptions, NPC behavior, what items/events are discoverable
- The world runs indefinitely — seasons cycle forever

## Tech Stack

| Layer | Choice |
|---|---|
| CLI client | Node.js |
| API server | Node.js + Express |
| Database | SQLite (`better-sqlite3`) — local on server |
| Intent parsing | Cloudflare AI |
| NPC dialogue / atmosphere | Venice AI (`VENICE_API_KEY`, OpenAI-compatible API) |
| Consequence reasoning | Claude (Anthropic SDK) |
| Deployment | Local server (Sleeper), PM2 for process management |
